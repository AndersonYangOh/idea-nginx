Directive can have multiple meanings. Each variant is separated with horizontal line

syntax: log_not_found [on|off]

default: log_not_found on

context: http, server, location

The directive enables or disables messages in error_log about files not found on disk.


Module: NginxHttpCoreModule

Syntax: log_not_found on | off

Default: on

Context: location

Enables or disables logging of requests that resolve into 404 status code. Can be used to prevent nginx from logging requests for the missing robots.txt and favicon.ico files. Example: 

location = /robots.txt {
  log_not_found  off;
}

Module: NginxHttpMainModule
