Directive can have multiple meanings. Each variant is separated with horizontal line

ssl_session_timeout

syntax: ssl_session_timeout time

default: ssl_session_timeout 5m

context: http, server

This directive defines the maximum time during which the client can re-use the previously negotiated cryptographic parameters of the secure session that is stored in the SSL cache.


Module: HttpSslModule

ssl_session_timeout

syntax: ssl_session_timeout time

default: 5m

context: mail, server

Assigns the time during which the client can repeatedly use the parameters of the session, which is stored in the cache.


Module: MailSslModule
