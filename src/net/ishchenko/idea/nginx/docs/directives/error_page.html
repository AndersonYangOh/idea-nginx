syntax: error_page

default: no

context: http, server, location, if in location

The directive specifies the URI, which will be showed for the errors indicated.

Example: 

error_page   404          /404.html;
error_page   502 503 504  /50x.html;
error_page   403          http://example.com/forbidden.html;
error_page   404          = @fetch;

Furthermore, it is possible to change the

error_page 404 =200 /.empty.gif;

If an erroneous answer is processed by the proxied or FastCGI server and this server can return the different answer codes, for example, 200, 302, 401 or 404, then it is possible to hide the

error_page   404  =  /404.php;

If you wish to return the error

error_page   404  /404.php;
